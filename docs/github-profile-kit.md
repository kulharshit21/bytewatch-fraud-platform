## Social preview strategy

Use one image that makes the repo understandable without reading the README:

- prefer the real product surface over abstract marketing art
- the shipped `github-social-preview.png` is a clean crop from the live `Overview` page
- it shows the analyst console, live status, business counts, producer controls, and product tone immediately
- keep the image dark, high-contrast, and minimal so it still reads in GitHub cards

## Quick apply checklist

1. Open the repo on GitHub.
2. Go to `Settings` -> `General` -> `Social preview`.
3. Upload `docs/assets/github-social-preview.png`.
4. Keep the repo description aligned with the text above.
5. Use the primary pinned repo caption on your profile.
6. Paste one of the About text options into your GitHub profile bio.

## Honest note

GitHub repo description and topics can be updated via CLI, but social preview image upload is still easiest through the GitHub web settings page. The asset is ready in-repo so you do not need to design it later.
